# 用例评审

## 适用场景

用户提供已有用例（表格/列表/脑图/文档），需要：找缺口、指出问题、自动修订用例、补充新增用例

## 规则依赖

开始评审前读取 `.cursor/skills/_shared/core-rules.md`，重点关注 §1 §2 §4 §5

## 执行流程（AI必读）

### Step 1: 问题识别（三级分类）
逐条检查用例，按优先级分类：
- **阻塞级**：字段缺失/期望模糊/无法执行 → 必须修复
- **重要级**：追溯缺失/冒烟错误/优先级不当 → 建议修复
- **一般级**：标题优化/描述改进 → 可选优化

### Step 2: 修订用例
- **修订已有用例**：补齐9字段、优化期望结果、修正冒烟标记
- **新增补缺口用例**：覆盖遗漏的TP/风险场景

### Step 3: Re-check复检
逐条验证关键Gate：
- ✓ 9字段齐全？（id/title/优先级/类型/执行端/前置/步骤/期望/冒烟）
- ✓ 期望结果可判定？（无"正常""成功"等模糊词）
- ✓ TP映射清晰？（可追溯到TP/TI）
- ✓ 冒烟规则正确？（P0∩主路径正向）

### Step 4: 回归集建议
- **P0必回归**：所有P0用例（含判定依据）
- **P1推荐回归**：变更影响范围内的用例（含影响分析）

## 快速口诀

| 维度 | 口诀 |
|------|------|
| **问题三级** | 阻塞（字段缺失/期望模糊）→ 重要（追溯缺失/冒烟错误）→ 一般（标题优化） |
| **必检三点** | 9字段齐全 + 期望可判定 + TP映射清晰 |
| **回归依据** | P0必回归（全部）+ P1推荐回归（变更影响+风险+历史缺陷） |
| **冒烟规则** | P0 ∩ 主路径正向（排除异常/边界/权限/并发） |

**评审维度速查**：
- **覆盖性**（有PRD/TD时）：RTM追溯 + 5类风险场景覆盖
- **可执行性**（必检）：9字段 + 前置4类 + 期望可判定
- **质量维护**：标题唯一 + 优先级合理 + 回归可抽取

> 优先级判定、冒烟规则详见 [core-rules.md](../_shared/core-rules.md)

## 输出结构

按以下5个章节输出评审报告：

1. **Issues（问题清单）** - 三级分类（必改/建议/可选），含问题点→影响→改法
2. **Patched cases（修订后用例集）** - 修订用例+新增用例（9字段齐全）
3. **Re-check（复检）** - 逐条验证字段/期望/TP映射/冒烟规则
4. **建议回归集** - P0必回归+P1推荐回归（含依据）
5. **待澄清清单** - 【未澄清】标记+Q-xxx问题

## 评审要点

**常见问题**：
- **阻塞级**：字段缺失、前置条件不具体（"有账号"）、期望模糊（"登录成功"）
- **重要级**：追溯缺失、冒烟错误（异常场景标为冒烟）、优先级不当
- **一般级**：标题优化、描述改进

**修订策略**：
- 补齐9字段（id/title/优先级/类型/执行端/前置/步骤/期望/冒烟）
- 前置条件包含4类：账号权限/配置开关/数据准备/环境依赖
- 期望结果可判定：状态码/字段值/UI文本/数据变化
