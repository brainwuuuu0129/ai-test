# TD 分析参考手册

## 1. 一致性与可靠性策略

| 策略 | 关键信息（缺失标注【未澄清】） | 验证要点 |
|-----|------------------------------|---------|
| **事务边界** | 事务范围/失败回滚/跨服务事务方案 | 全成功数据一致/部分失败全回滚/超时有补偿 |
| **幂等策略** | 幂等键/幂等方式/有效期/重复请求行为 | 重复请求仅执行一次/过期后可重复/并发下有效 |
| **重试策略** | 重试次数/间隔/重试场景/副作用 | 可重试场景正确重试/与幂等配合无重复副作用 |
| **补偿回滚** | 补偿场景/策略/失败兜底 | 补偿成功数据恢复一致/失败有告警和人工介入 |
| **并发冲突** | 冲突场景/处理方式/锁超时 | 并发写数据最终一致/锁正常释放无死锁 |

## 2. 可观测性清单

### 2.1 可观测性4要素

| 要素 | 必需内容 | 验证标准 |
|-----|---------|---------|
| **日志** | traceId + 业务ID + 结果 + 关键参数 | 关键操作有日志，可串联全链路，敏感信息脱敏 |
| **指标** | 成功率 + QPS + 延迟(P95/P99) + 错误率 | 指标可采集查询，准确反映系统状态 |
| **告警** | 成功率<95% / 错误率>5% / 延迟P95>500ms | 告警规则配置正确，包含必要上下文 |
| **链路** | traceId + spanId + 调用关系 + 耗时分布 | 链路完整无断点，异常可追溯到具体节点 |

### 2.2 可测性判定简表

常用技术验证手段及指引：

| 验证方式   | 适用验证内容                    | 说明与补充         |
|------------|-------------------------------|-------------------|
| 接口调用   | 功能/集成/幂等/重试             | 依赖真实或Mock环境 |
| 日志检查   | 操作/异常/补偿行为              | 需明确日志信息     |
| DB查询     | 数据一致性/状态变更             | 侵入性高，需连库   |
| 指标采集   | 性能/可用性/容量                | 需有指标上报       |
| 链路追踪   | 调用关系/耗时/异常定位           | 需有traceId等      |
| Mock注入   | 第三方依赖/异常注入              | 需Mock或沙箱策略   |
| 人工观察   | UI/流程/无自动化接口            | 标注【不可自动化】 |
| 信息缺失   | TD未说明验证手段                 | 标注【未澄清】     |

## 3. 接口契约与配置

| 类型 | 必须明确的信息 | 验证要点 |
|-----|--------------|---------|
| **接口契约** | 鉴权/入参出参/错误码/限流/熔断/超时 | 鉴权有效/校验完整/错误码准确 |
| **第三方集成** | 依赖服务/降级策略/Mock方案/超时重试 | 正常调用正确/异常降级有效 |
| **配置管理** | 功能开关/业务配置/默认值 | 配置可动态修改，修改生效及时 |
| **灰度回滚** | 灰度方式/比例/回滚方案 | 灰度用户可用/回滚后恢复 |

## 4. 典型TD场景示例

**幂等机制**：Redis SET NX + order_id，验证重复请求/并发/过期/故障降级

**重试补偿**：指数退避重试3次，补偿机制定时轮询，配合幂等无副作用

**熔断降级**：错误率>50%触发熔断，返回默认推荐，半开状态恢复

**限流验证**：Redis滑动窗口，1分钟3条/1小时10条，超限返回429

**并发写冲突**：乐观锁version字段，冲突返回409，客户端重试3次

> 以上为典型场景摘要。各场景的 TI 产出格式见 SKILL.md 核心示例（Step 4 TI清单）。

## 5. 边界Case处理

| 场景 | 检测方式 | 处理方式 | 示例 |
|------|---------|---------|------|
| **TD信息不完整** | 关键策略缺失/实现细节不明 | 不阻塞已确定部分，对缺失标注【未澄清】，给出最小澄清清单 | 见§4幂等机制示例 |
| **仅有代码无TD** | 提供代码文件而非TD文档 | 明确告知基于代码分析，不确定项标注【未澄清】，建议补充正式TD | 基于代码推断幂等策略 |
| **多系统集成** | 涉及多个服务/中间件 | 按系统拆分TI，标注依赖关系，给出联调建议 | TI-001依赖服务A，TI-002依赖服务B |
| **技术方案变更** | TD版本迭代 | 标注[新增]/[修改]/[废弃]，对比旧版本，给出回归建议 | TI-010 [修改]：幂等方式从DB改为Redis |

## 6. 常见问题FAQ

| 问题 | 检测方式 | 处理方式 | 示例 |
|------|---------|---------|------|
| **TI优先级不确定** | 不知如何判定P0/P1/P2 | 资金/数据安全/核心链路→P0；关键分支/高频→P1；其他→P2 | 支付幂等→P0，推荐降级→P1 |
| **可观测性不足** | 日志/指标缺失 | 三问：故障能否快速定位？系统能否实时监控？异常能否及时发现？不足标注【待补充】 | 缺少支付成功率指标 |
| **验证手段不明确** | 不知如何验证 | 参考§2.2可测性判定简表 | 幂等→接口调用，补偿→日志+DB |
| **TI粒度不确定** | 不确定拆分粒度 | 按技术验证点拆分，每个TI可展开为3-10个TC | 幂等机制拆为：重复/并发/过期/故障 |
| **依赖第三方无法测试** | 外部系统不可控 | 说明Mock方案或沙箱环境验证方式 | Mock支付网关返回超时 |

## 7. 常见TD模式检查表

**用法**：Step 2 分析每个 TD 模块时，主动识别下表模式并补充对应 TI。有信号 → 必须追加；TD 明确提及 → 至少出 1 个 P0 TI。

| 模式 | 触发信号（TD 中出现以下关键词） | 必须追加的 TI 验证点 |
|------|-------------------------------|-------------------|
| **Job / 定时任务** | Job、定时任务、Scheduler、cron | ① 幂等键存在，重复触发数据不重复；② 失败时有告警或补偿机制 |
| **异步操作** | async、异步、MQ、消息、EventBus | ① 失败后中间状态正确（不丢失、不卡死）；② 补偿/重试不产生重复副作用；③ 关键步骤有日志可追踪 |
| **枚举 / 类型变更** | 新增枚举值、废弃枚举、类型扩展、枚举变更清单 | ① 旧版本客户端反序列化不报错；② 新枚举对现有业务流程无影响；③ DB 存储兼容 |
| **强制校验** | "前端校验"、"客户端校验"、"APP 侧校验" | ① 后端接口独立校验（绕过前端直调返回正确错误码）；② 前后端校验规则一致 |
| **状态机** | 状态图、状态枚举、status 字段流转、合法流转 | ① 每条合法流转路径验证；② 非法流转被拒绝（返回具体错误码）；③ 并发流转幂等 |
| **计算逻辑改动** | totalXxx() 改动、计算方法变更、字段累加逻辑、新增类型参与计算 | ① 改动前后现有路径计算结果不变（回归）；② 新路径计算正确；③ 边界值（0、负数、极大值） |
| **改动现有接口** | "现有接口"、"改动"、"扩展"、"新增字段"、"行为变更" | ① 现有调用方传参不变时响应不变（回归）；② 新增字段/参数缺省时行为合理（默认值不影响现有逻辑） |
| **RBAC / 权限控制** | "管理员权限"、"角色校验"、"鉴权"、"adm-api"、"需要管理员" | ① 无权限 Token 调用返回 401/403；② 低权限角色调用高权限接口被拒绝；③ 接口层独立鉴权（不依赖前端隐藏入口） |
