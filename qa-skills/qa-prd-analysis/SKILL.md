# PRD 分析（面向测试）

## 适用场景

用户提供PRD（Confluence/PDF/截图/文本），需要产出测试化分析报告

## 规则依赖

开始分析前读取 `.cursor/skills/_shared/core-rules.md`，重点关注 §1 §2 §3 §5 §6

## 执行流程（AI必读）

### Step 1: 事实摘录
从PRD抽取可引用事实（范围/角色/流程/规则） → 建立FACT-xxx清单，每条标注来源
→ **特别标记**：含数值/长度/数量约束的字段（如价格/数量/日期/字符长度），建立"边界字段清单"供Step 3 TP直接引用

### Step 2: 测试化分析
基于事实摘录产出10个章节内容（见下方输出模板）：
- 明确范围In/Out、角色权限、流程状态机
- 提炼业务规则（含边界/默认值/失败语义）
- 识别5类风险场景，每类产出 RISK-xxx（原因→影响→验证方式）→ 填入第8章

### Step 3: TP清单产出
逐条产出TP-xxx，每个TP包含：
- id/name/source（可追溯到FACT或REQ）
- priority（P0/P1/P2，含判定依据）
- 执行端（Web/App/API/混合，来自PRD系统路径说明）
- 验证方式（功能测试/自动化/手工等）

> 拆分粒度指南：按功能点/业务规则拆，不按正向/异常拆（正向/异常是TC层面）→ 详见 [reference.md §2.2](references/reference.md#22-tp拆分粒度指南)

### Step 4: 质量检查
- ✓ 范围In/Out清晰？验收标准可判定？
- ✓ 每个TP可追溯或标【未澄清】？
- ✓ 5类风险场景覆盖？
- ✓ 冒烟=P0且主路径正向？

> 详细流程、PRD类型决策树、TP拆分粒度指南见 [reference.md §1](references/reference.md#1-输入处理流程)

## 快速口诀

| 维度 | 口诀 |
|------|------|
| **分析方法** | 事实先行：先摘录再分析，不做假设 |
| **TP追溯** | REQ → TP → TC，链路清晰可追溯 |
| **风险5类** | 功能/权限/数据/兼容/可观测 |
| **优先级** | 资金/安全/合规优先 → P0 |
| **冒烟规则** | P0 ∩ 主路径正向流程 |

**质量门槛**：
- ✓ 范围In/Out清晰，验收标准可判定
- ✓ 每个TP可追溯或标【未澄清】
- ✓ 5类风险场景必须覆盖（不在范围需说明）

> 优先级判定、冒烟规则详见 [core-rules.md](../_shared/core-rules.md)

## 输出结构

按以下10个章节输出分析报告：

1. **背景/目标** - 需求背景和目标
2. **范围** - In scope/Out of scope/约束
3. **角色与权限** - 角色｜入口/端｜允许操作｜禁止操作
4. **流程与状态机** - 主流程/关键分支/状态机
5. **业务规则** - 规则+边界/默认值+失败语义
6. **数据口径与字段** - 关键字段/校验/埋点
7. **验收标准** - AC1, AC2...（可判定）
8. **风险清单** - RISK-xxx（原因→影响→验证方式）
9. **测试点清单** - TP-xxx（id/name/source/priority/验证方式）
10. **待澄清清单** - 【未澄清】标记+Q-xxx问题+QA回答区

## 核心示例

**输入PRD片段**：
```
用户登录功能：
- 用户名：邮箱或手机号
- 密码：6-20位，包含字母和数字
- 密码错误：提示"用户名或密码错误"
- 密码连续错误3次：锁定账号30分钟
```

**Step 1: 事实摘录**

| FACT-ID | 内容 | 来源 |
|---------|------|------|
| FACT-001 | 用户名可以是邮箱或手机号 | PRD §1.1 |
| FACT-002 | 密码：6-20位，包含字母和数字 | PRD §1.1 |
| FACT-003 | 密码错误 → 提示"用户名或密码错误" | PRD §1.1 |
| FACT-004 | 密码连续错误3次 → 锁定账号30分钟 | PRD §1.2 |

**Step 3: TP清单产出**

| TP-ID | 测试点 | 来源 | 优先级 | 判定依据 |
|-------|-------|------|--------|---------|
| TP-001 | 验证邮箱/手机号登录 | FACT-001 | P0 | 核心链路主流程 |
| TP-005 | 验证密码错误提示 | FACT-003 | P0 | 安全相关 |
| TP-006 | 验证账号锁定机制 | FACT-004 | P0 | 安全相关 |

**待澄清示例**：
```
【未澄清：密码"包含字母和数字"是必须同时包含，还是可以只含其一？｜影响：TP-004｜需要：产品确认密码组成规则】
```
