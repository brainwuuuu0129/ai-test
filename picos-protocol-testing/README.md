# PicOS Network Protocol Test Case Generator

AI-driven test case generation for Pica8 PicOS network switch protocol implementations.

## Overview

This project uses AI (Cursor Agent + custom Skills) to automatically generate comprehensive, executable functional test cases for PicOS network protocols. Test cases are generated by cross-referencing **RFC standards** with **PicOS vendor documentation**, ensuring both protocol compliance and vendor-specific CLI accuracy.

## Quick Start

### Prerequisites

- [Cursor IDE](https://cursor.sh) with Agent mode
- Python 3.9+
- Internet access (for fetching PicOS Wiki docs)

### Generate Test Cases for a New Protocol

1. **Prepare documentation**

```bash
# Download the RFC
python scripts/fetch_picos_docs.py --protocol bgp

# Or manually place files:
# - RFC document в†’ docs/rfcs/
# - PicOS config guide в†’ docs/picos/
```

2. **Generate test cases** (in Cursor Agent mode)

```
@generate-protocol-test-cases

Protocol: BGP
RFC: docs/rfcs/rfc4271.txt
Vendor Guide: docs/picos/bgp-config-guide.md
Platform: PicOS 4.6
Test Type: Functional
Output Language: English
```

3. **Export to PDF**

```bash
python scripts/md2pdf.py test-cases/bgp/bgp-functional-test-cases.md
```

## Test Case Generation Process

The AI Skill follows a 4-stage pipeline:

```
Stage 1: Protocol Understanding & Feature Mapping
    в†“ Read RFC + PicOS docs в†’ Feature-Module Mapping Table
Stage 2: Test Point Extraction
    в†“ FSM / Packet Field / Message Sequence / Protocol-Specific modeling
Stage 3: Test Case Generation
    в†“ FS.COM table format with real PicOS CLI commands
Stage 4: Coverage Verification & Gap Filling
    в†“ Coverage matrix + supplementary cases
```

## Test Case Output Format (FS.COM Table)

Each test case is output as a vertical table with 13 fields:

| Field | Description |
|-------|-------------|
| Test Name | Module ID + sequence + title |
| Purpose Of The Test | What is being verified, with RFC reference |
| Test Topo & Precondition | Topology diagram + preconditions + full CLI configuration |
| Test Procedure | Step-by-step actions (action verbs only) |
| Expected Results | Observable outcomes from show commands / state changes |
| Automated or Not | Automation status |
| Related Scripts | Automation script path |
| Level | P0 / P1 / P2 |
| Hardware Model | Target switch model |
| Version | PicOS software version |
| Actual Results | Filled during execution |
| Test Results | Pass / Fail (filled during execution) |
| Remark | RFC reference, notes |

## Utility Scripts

### fetch_picos_docs.py

Fetches PicOS configuration guides from the Pica8 Wiki and saves them locally.

```bash
python scripts/fetch_picos_docs.py --protocol ospf --version 4.6
python scripts/fetch_picos_docs.py --protocol bgp --version 4.6
```

### md2pdf.py

Converts markdown test case files to styled PDF documents.

```bash
python scripts/md2pdf.py test-cases/ospf/ospf-functional-test-cases-mod1-3.md
python scripts/md2pdf.py test-cases/ospf/ospf-functional-test-cases-mod4-6.md
```

### merge_test_cases.py

Merges multiple module files into a single document.

```bash
python scripts/merge_test_cases.py --protocol ospf --output test-cases/ospf/ospf-complete.md
```

## Methodology

This project's approach is inspired by the **NeTestLLM** multi-agent framework for network protocol testing:

1. **Hierarchical Protocol Understanding** вЂ” Break down RFCs into feature modules
2. **Multi-Model Test Point Extraction** вЂ” FSM, Packet Field, Message Sequence, Protocol-Specific
3. **Vendor-Specific CLI Binding** вЂ” Every command comes from real PicOS documentation
4. **Quality Gate Verification** вЂ” 3-level check (executability, coverage, maintainability)
