## §1 输入处理规则

### 1.1 规格书解析优先级

1. **必须提取**（缺失则无法生成用例，需向用户询问）：
   - Form Factor（SFP / SFP+ / QSFP28 / QSFP-DD / AOC / DAC）
   - 速率（1G / 10G / 25G / 40G / 100G / 400G）
   - DDM 支持（Yes / No）

2. **应当提取**（缺失则在 Remark 标注 `[未澄清：<字段名>]`）：
   - 波长 / 接头类型 / 传输介质（单模 SMF / 多模 MMF / 铜缆）
   - TX/RX 功率范围（dBm）
   - 告警阈值表（High Alarm / High Warn / Low Warn / Low Alarm）
   - 工作温度范围

3. **可选**（有则利用，无则跳过）：
   - 最大传输距离
   - 支持的平台兼容列表
   - FEC 要求

### 1.2 CLI 手册解析规则

- 每条命令必须来自手册原文，**禁止拼造命令**
- 提取时注明命令的权限要求（user/admin/enable）和适用上下文（接口模式/全局模式）
- 若手册未提供某操作的命令，在用例步骤中标注 `[需厂商确认CLI]`，不影响其他用例输出

## §2 四个维度测试点展开

### 2.1 兼容性（COMPAT）

| 测试点 | 说明 | 优先级 |
|---|---|---|
| 模块识别 | 平台能否读取 Vendor Name / Part Number / Serial Number | P0 |
| 链路 UP | 双端插入同型号光纤/线缆后链路建立 | P0 |
| 端口状态 | 链路 UP 后 speed / duplex / media type 符合规格 | P0 |
| 速率协商 | 支持多速率模块时验证各速率档位 | P0 |
| 跨厂商互通 | A 厂交换机 + B 厂模块（若适用）| P1 |
| 端口类型匹配 | 将模块插入不匹配的端口槽（如 QSFP 插 SFP 槽）报错处理 | P1 |

### 2.2 DDM 监控（DDM）

> AOC/DAC 通常不支持 DDM，若 DDM = No，本维度全部用例标注 `N/A (无DDM支持)`。

| 测试点 | 说明 | 优先级 |
|---|---|---|
| 温度读数 | 实时温度在规格书工作范围内可正常读取 | P0 |
| 供电电压读数 | 电压读数正常（典型 3.3V ±5%）| P0 |
| TX 功率读数 | 发射光功率在规格书 TX Range 范围内 | P0 |
| RX 功率读数 | 接收光功率在规格书 RX Range 范围内 | P0 |
| 偏置电流读数 | 偏置电流正常（典型 < 100mA）| P0 |
| High Alarm 触发 | 某项指标超高告警阈值时平台产生告警日志 | P1 |
| Low Alarm 触发 | 某项指标低于低告警阈值时平台产生告警日志 | P1 |
| Warn 级别区分 | High/Low Warn 与 Alarm 告警级别在日志中可区分 | P1 |
| 告警恢复 | 指标恢复正常范围后告警清除 | P1 |

### 2.3 链路性能（PERF）

| 测试点 | 说明 | 优先级 |
|---|---|---|
| 误码率（BER）| 发送流量后统计误码率 ≤ 规格书要求 | P1 |
| 错误帧计数 | 持续流量下错误帧计数不异常增长 | P1 |
| 光功率预算验证 | 插入不同长度光纤，RX 功率在 link budget 范围内 | P1 |
| 最大传输距离 | 在规格书额定距离下链路可正常建立 | P1 |

### 2.4 热插拔 & 异常（HOTSWAP）

| 测试点 | 说明 | 优先级 |
|---|---|---|
| 热插 | 系统运行中插入模块，链路可正常建立，无系统重启 | P0 |
| 热拔 | 系统运行中拔出模块，端口状态变 down，无系统崩溃 | P0 |
| 反复热插拔 | 连续插拔 ≥3 次，系统稳定，每次均可正常识别 | P0 |
| 非认证/非法模块 | 插入未认证模块时平台给出告警或拒绝，不崩溃 | P1 |
| 告警恢复 | 热拔产生的告警在重新插入并链路 UP 后自动清除 | P1 |

## §3 DDM 5 项指标规范

当规格书未提供阈值时，以下为典型参考范围（**仅供参考，优先使用规格书数据**）：

| 指标 | 单位 | 典型工作范围 | High Alarm | Low Alarm |
|---|---|---|---|---|
| 温度 | ℃ | 0 ~ 70（商业级）| +90 | -10 |
| 供电电压 | V | 3.135 ~ 3.465（3.3V ±5%）| 3.6 | 3.0 |
| TX 光功率 | dBm | 模块规格书 TX Range | TX Max + 1dBm | TX Min - 1dBm |
| RX 光功率 | dBm | 模块规格书 RX Range | RX Max + 1dBm | RX Min - 3dBm |
| 偏置电流 | mA | 0 ~ 100（典型）| 110 | 1 |

**读数格式要求**：Expected Results 中的 DDM 数值必须写成范围，例如：
- 正确：`温度读数在 0~70℃ 范围内`
- 错误：`温度读数正常`（不可判定）

## §5 优先级判定规则

| 场景 | 优先级 | 理由 |
|---|---|---|
| 模块识别、链路 UP | P0 | 核心路径，失败则所有后续测试无法进行 |
| DDM 5 项指标读数 | P0 | 监控基础功能，生产环境强依赖 |
| 热插 / 热拔 / 反复插拔 | P0 | 数据中心常见操作，稳定性要求高 |
| DDM 阈值告警触发与恢复 | P1 | 重要但需额外配置，不影响基本链路 |
| 链路性能（BER/错误帧）| P1 | 需要流量生成器，环境要求高 |
| 跨厂商互通 | P1 | 依赖外部模块可用性 |
| 极端条件（最大距离/边界电压）| P2 | 边界验证，非日常覆盖必需 |

## §6 常见问题（FAQ）

**Q1: AOC/DAC 不支持 DDM，怎么处理？**
在 DDM 维度的每条用例的 `Remark` 字段标注：`N/A：AOC/DAC 不支持 DDM`，用例 Level 改为 `Skip`。兼容性和热插拔维度正常生成。

**Q2: 规格书没有提供告警阈值表，怎么处理？**
使用 §3 典型参考范围，在 Remark 标注：`阈值来自行业典型值（§3），非规格书数据，执行前需与厂商确认`。

**Q3: 用户未提供 CLI 手册，只有规格书怎么办？**
停止生成，向用户说明：`缺少平台 CLI 手册，无法填写测试步骤中的 show 命令。请提供目标平台的 transceiver/DDM 查询命令文档后继续。`

**Q4: 模块支持多速率（如 1G/10G 自适应），怎么生成用例？**
每个速率档位单独生成一条 COMPAT 用例，Test Name 格式：`COMPAT-001 SFP+ 10G 链路建立`、`COMPAT-002 SFP+ 1G 降速链路建立`。

**Q5: 如何处理非认证模块测试？**
归入 HOTSWAP 维度，Test Name 格式：`HOTSWAP-004 非认证模块插入处理`。Expected Results：平台输出告警日志 `[关键字：unauthorized / unsupported]`，端口可配置为 error-disabled 或 up（取决于平台策略），系统不崩溃。
